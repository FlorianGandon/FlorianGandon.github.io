Amélioration d'une base d'un jeux RPG en Golang

janv. 2023

Présentation :

Ce projet était un travail de mon université à faire en duo, je l’ai fait avec Armel Cloarec. Au début on nous a fournit une base d’un jeu RPG avec des tuiles de 16 par 16 pixels, avec un personnage.

Dans la première partie, nous dûmes récupérer le terrain à partir sur un fichier texte, le stocker dans un liste de liste et de l’afficher sur la fenêtre. Ensuite il fallait créer un package où la carte était enregistré dans un Quadtree.

La deuxième grande partie était une série de demande à implémenter sur la base tout en permettant l’activation et la désactivation de ce module dans un fichier configuration.

Les modules complémentaire sont :

  • le déplacement de la caméra pour qu’elle soit fluide (sinon elle se déplace de case en case)
  • StopCamera : la caméra n’affiche pas des zones en dehors de la carte sauf dans le cas où le carte est plus petite que l'affichage alors la carte sera au centre.
  • RoundEarth : la carte se répète à l'infini donc le joueur ne peut pas la quitter
  • RandomGeneration : génère une carte de manière aléatoire et infinie
  • la sauvegarde d’une carte généré aléatoirement
  • ConnectedTile : les textures s’adapte des tuiles autours d’elle pour faire des contours plus naturels.
  • ActiveBigStep : qui permet de se déplacer plus vite avec CTRL + les flèches directionnelles.
  • BetterCharacter : offre un affichage du joueur meilleur
  • BetterBlocking : le joueur ne peut pas marcher sur l’eau et les murs (sauf en debug mode)
  • ActivePortal : permet de créer deux portail pour se téléporter
  • ActiveParticlesPortal : ajoute des particules au portails
  • ActiveScrolling : change la taille d'affichage avec la molette de la souris

Les apprentissages :

Tous d’abord pour la génération aléatoire nous avons modifier une classe du programme de base ce qui a empêcher les testes du prof de fonctionner. Ce qui nous montre qu’un implémentation d’un bout de code ne doit pas venir modifier le code existant car cela peut casser des testes automatiques et empêcher le programme de fonctionner normalement.
